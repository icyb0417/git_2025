# 课程学习总结

## 一、认识git与github

* git是现代软件开发必不可少的版本控制工具，而github则是一个至关重要的基于git版本控制系统的代码托管平台。掌握git与github的使用，能够大大提高团队的开发效率和代码质量。

## 二、对git的理解

* 版本控制系统不仅仅是简单的copy，而是完整地记录了项目代码的演变过程，让我们能够更高效地管理代码，更改代码。

### git的核心命令

* git init #初始化仓库
* git clone #获取远程仓库
* git add #添加文件到暂存区
* git commit #将暂存区的更改提交到本地库
* git status #查看仓库的提交状态
* git log #查看提交历史
* git branch #查看所有分支
* git pull #合并远程仓库代码
* git push #上传代码

## 三、对github的理解

* github的核心功能在于代码管理、团队协助开发、开源社区，为开发者提供git仓库的托管服务，提供一个让开发者共享代码的完美场所，帮助其高效率、高品质地进行代码编写。

### github的核心内容

#### 1.远程仓库管理

* 将本地仓库与github远程仓库通过git remote命令进行关联，并通过git pull和git push命令进行代码的同步，实现本地与远程仓库的数据互通

#### 2.Fork和Pull Request

##### 请求过程

* 首先Fork仓库到个人仓库，接着Clone仓库里面的SSH，通过创建新分支来修改仓库内容，修改完毕后commit和push到自己本地的仓库，接着创建pull request请求。

##### 价值作用

* 通过Pull Request，既可以向原项目贡献自己创作的新代码，原仓库所有人也可以审查我的新代码，完美地体现了开源协作的作用，整个过程简单高效，贡献代码者不需要获取原项目的写入权限，原仓库所有人也不会因新代码而破坏了自己原有的代码仓库。

## 总结

* 课程学习到了如何使用git和github，并掌握了核心的pull request技能，一个看似十分简单的拉取请求的过程，却能够完美解决很多开发者的问题，对于git和github还需要不断掌握不断探索，不断提高对其的理解。
