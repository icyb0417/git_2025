# 课程学习总结

## 一、学习初期：从陌生到入门
最开始接触这门课程时，我对「Git版本管理」的概念几乎是空白的，尤其是 **分支创建、版本回退** 这些操作，总觉得逻辑绕不清楚。

比如第一次尝试`git commit`时，经常忘记写提交信息，导致命令报错；对着终端里的英文提示，只能一点点查翻译和教程。


## 二、学习中期：实践中摸索方法
后来跟着课程的实操案例练习，我慢慢找到了节奏：
- 每学一个新命令（比如`git branch`、`git merge`），都会自己建一个小仓库测试；
- 遇到问题先截图保存错误信息，再去课程文档或GitHub帮助页找解决方案；
- 和同学组队做了一个「笔记仓库协作」的小练习，第一次用`git push`和`pull`完成了多人同步。


## 三、学习收获：技能与思维的双重提升
通过这门课，我不仅掌握了：
1. Git的基础操作（提交、分支、合并）；
2. GitHub Fork + PR的协作流程；
更重要的是养成了「**先备份、再修改**」的习惯——每次改文件前，都会用`git status`确认状态，避免误操作丢失内容。


## 四、后续计划
接下来我打算用Git管理自己的「学习笔记仓库」，同时尝试参与开源项目的PR提交，把课程里学的内容真正用起来。
